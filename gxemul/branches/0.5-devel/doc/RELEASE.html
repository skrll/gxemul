GXemul: Release notes

Release notes for GXemul 0.5.0

Copyright (C) 2003-2009 Anders Gavare


GXemul is a framework for full-system computer architecture emulation/simulation.


The changes between release 0.4.7 and 0.5.0 are:

  • A complete redesign of the emulation framework has begun. The new framework is written in C++, and should hopefully be a more robust and extensible platform than the old one was.

    Some of the highlights:

    • A much more general view of what an emulated "component" is than before, allowing for more complex emulation setups, for example mixing different CPU architectures within a machine.
    • Unit tests for most of the functionality.

    (Since this is such a huge undertaking, only very few actual emulation modes have been reimplemented to use the new framework. GXemul 0.4.x is of course still available for download, if you need emulation modes or features from those versions.)

  • Initial I18N support. Apart from English, there is now also a Swedish translation.

  • There is now (finally) a 'make install' Makefile target.

To build the emulator, run the configure script, and then run make. This should work on most Unix-like systems, with few or no modifications to the source code.

Regarding files in the src/include/thirdparty directory: these are imported from other sources, such as NetBSD. The license texts usually say that "All advertising materials mentioning features or use of this software" must display acknowledgements. Even though I do not feel that I mention features or use of the header files (the "software") in any advertising materials, I am still very grateful for the fact that these people have made their files available for re-use, as Free software, so regardless of legal requirements, I guess thanking them like this is in order:
